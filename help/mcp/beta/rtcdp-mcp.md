# Real-Time CDP MCP （Beta） {#rtcdp-mcp}

Adobe Real-Time CDPのMCP統合を利用すれば、API呼び出しを記述したり、製品画面を移動したりすることなく、平易な言語のプロンプトを使用して、オーディエンス、宛先、アクティベーションの正常性をクエリできます。 この統合は、Adobe Real-Time CDPとAdobe Real-Time CDP B2B editionの両方のお客様にサービスを提供し、MCP互換クライアントからサポートされているReal-Time CDP データとワークフローを調べるための会話型の方法を提供します。 このガイドでは、統合がどのように機能するのか、何ができるのか、どのように始めればよいのかなどについて説明します。

>[!AVAILABILITY]
>
>Real-Time CDP MCPはBetaにあります。 機能とドキュメントは変更される可能性があります。 Real-Time CDP MCP サーバーは、サポートされているMCP クライアントおよびアプリプラットフォームにユーザーがインストールおよび設定する&#x200B;**リモート HTTP トランスポート サーバー**&#x200B;として配布されます（例：[!DNL Claude]、[!DNL ChatGPT]、[!DNL Claude Code]、[!DNL Codex]、[!DNL Cursor]、または[!DNL VS Code]）。 認証は、**ブラウザーベースのログインフロー**&#x200B;を通じて処理されます。クライアントが最初にサーバーに接続すると、デフォルトブラウザーが開き、Adobeの資格情報でログインしてアクセスを許可できます。 このBeta プログラムにアクセスするには、Adobe担当者にお問い合わせください。

## モデルコンテキストプロトコルとは何ですか？ {#mcp-overview}

マーケティング、データ、顧客体験の各チームは、日々の作業を合理化するために、チャットベースのアプリケーションや開発者ツール（Anthropic Claude、OpenAI ChatGPT、Cursor、Microsoft Copilot Studioなど）への依存度を高めています。 これらのアプリケーションは、**Model Context Protocol （MCP）**&#x200B;をサポートしています。これは、アプリケーションがバックエンドツールを大規模言語モデル（LLM）に統一された方法で公開できるようにするオープンスタンダードです。

Real-Time CDPは、MCP互換アプリケーション内でオーディエンス、宛先、アクティベーションの操作を直接表示するMCP サーバーを提供するようになりました。 Real-Time CDPとMCPを連携することで、Adobe Experience Platform REST APIに対してクエリを作成したり、複数のUI画面を操作したりすることなく、様々なペルソナが同じセグメンテーションデータやアクティベーションデータを利用して共同作業をおこなうことができます。 顧客は自分の意図を対話的に説明でき、LLMが適切なMCP ツールを呼び出せます。

## 主な機能 {#mcp-capabilities}

Real-Time CDP MCP サーバーは&#x200B;**読み取り専用**&#x200B;の監視およびトリアージ サーフェスです。 AI アシスタント内で、クエリを記述したり製品画面を移動したりすることなく、オーディエンス、宛先、ソース、ID、プロファイル解決をまたいで、取得APIを平易な言語で回答として公開します。 MCP サーバーを通じてデータを作成、変更、または削除することはできません。

>[!IMPORTANT]
>
>現在のBetaのすべてのツールは&#x200B;**読み取り専用**&#x200B;です。 オーディエンス、宛先、データフローの作成、アクティブ化、更新、削除などの書き込み操作はサポートされていません。

Beta リリースには、次の18のツールが含まれています。

| ツール | 説明 |
| --- | --- |
| `search_audiences` | 名前、エンティティの種類、ライフサイクルの状態、ID名前空間、オリジンごとにオーディエンスをリスト化して検索できます。 |
| `preview_audience_membership` | オーディエンスとして保存する前に、PQLまたはSDD セグメント式のサイズを見積もります。 |
| `inspect_audience_evaluation_jobs` | セグメント評価のジョブレコードを取得して、バッチオーディエンスが更新されない理由を診断したり、最近の評価履歴を確認したりできます。 |
| `inspect_audience_export_jobs` | オーディエンス書き出しジョブレコードを取得して、書き出しが完了したことを確認するか、失敗の詳細を表示します。 |
| `search_destination_connectors` | プラットフォームで使用可能な宛先コネクタの種類（例：[!DNL Amazon S3]、[!DNL Google Ads]、[!DNL Salesforce] CRM）を一覧表示します。 |
| `search_destination_accounts` | 認証済み宛先アカウントのリスト – 宛先コネクタタイプの設定済みインスタンス。 |
| `search_destination_input_connections` | 書き出されるオーディエンスまたはデータセットなど、宛先フローのExperience Platform側の入力を取得します。 |
| `search_destination_output_connections` | 宛先フローの外部エンドポイント（ターゲットパス、ファイル形式、配信設定）を取得します。 |
| `search_destination_flows` | 状態、マッピング、スケジュールなど、設定済みの宛先アクティベーションフローを一覧表示して調べます。 |
| `inspect_flow_runs` | ソースおよび宛先フローの実行履歴を取得します。実行ごとに、ステータス、タイミング、レコード数、エラーの詳細を取得します。 |
| `search_source_connectors` | プラットフォームで使用可能なソースコネクタタイプを一覧表示します。 |
| `search_source_accounts` | 認証されたソースアカウントのリスト – ソースコネクタタイプの設定済みインスタンス。 |
| `search_source_input_connections` | ソースフローのデータ選択レイヤー（アカウントから取得されたもの）を取得します。 |
| `search_source_output_connections` | 取り込まれたデータが格納されるソースフローのExperience Platform データセットの宛先を取得します。 |
| `search_source_flows` | 状態、マッピング、スケジュールなど、設定済みのソース取り込みパイプラインを一覧表示および検査します。 |
| `search_identity_namespaces` | Adobe標準のネームスペースとカスタムのネームスペースの両方で、サンドボックスにID名前空間の定義をリストアップします。 |
| `search_merge_policies` | プロファイルフラグメントからリアルタイム顧客プロファイルを組み立てる方法を制御する、結合ポリシーレコードのリスト。 |
| `search_organizations` | 認証済みユーザーがアクセスできるAdobe組織を一覧表示します。 |

## アクセスとイネーブルメント {#mcp-access}

>[!AVAILABILITY]
>
>Real-Time CDP MCP サーバーはBetaにあり、セルフサービス登録用に開いていません。 アクセスは招待状によってのみ行われ、接続する前にAdobe組織に明示的に許可リストに加えるする必要があります。

アクセスをリクエストするには：

1. Adobeのアカウント担当者（カスタマーサクセスマネージャー、テクニカルアカウントマネージャー、アカウントエグゼクティブ）に連絡し、Real-Time CDP MCP Beta プログラムへの関心を伝えます。
2. Adobeの担当者が製品チームと連携して、利用条件を検討し、組織IDを有効にします。
3. 有効になると、Adobe担当者がアクセスを確認し、その他のオンボーディング資料を提供します。

>[!NOTE]
>
>Real-Time CDP MCP サーバーに接続できるのは、明示的に有効になっている組織のみです。 イネーブルメントの前に接続しようとすると、認証エラーが発生します。

## 前提条件 {#mcp-prerequisites}

Real-Time CDP MCP サーバーをMCP クライアントに接続する前に、次の点を確認してください。

* アクティブなReal-Time CDP ライセンスがあります。
* お客様のAdobe組織は、Adobe担当者によってBeta プログラムに対して有効になっています（[ アクセスと有効化](#mcp-access)を参照）。
* サポートされているMCP クライアント （[!DNL Claude]、[!DNL ChatGPT]、[!DNL Claude Code]、[!DNL Codex]、[!DNL Cursor]、または[!DNL VS Code]など）にアクセスできます。
* 組織IDと、クエリするサンドボックスの名前があります。
* オーディエンス、宛先、フローサービスエンティティを表示するために必要な権限をAdobe Experience Platformに持っています。

## Real-Time CDP MCP サーバーへの接続 {#mcp-connect}

Real-Time CDP MCP サーバーエンドポイントは次のとおりです。

```
https://rtcdp-mcp.adobe.io/mcp
```

サーバーは、**ブラウザーベースのAdobe サインインフロー**&#x200B;を使用して&#x200B;**リモート HTTP （ストリーマブル HTTP） トランスポート**&#x200B;を使用します。 すべてのクライアントで、設定パターンは同じです。

1. サーバーURLを追加：`https://rtcdp-mcp.adobe.io/mcp`
2. 接続を保存または有効にします。
3. クライアントが初めてツールを呼び出すときに、**ブラウザーベースのAdobe ログイン**&#x200B;を完了します。
4. 各セッションの開始時に`imsOrgId`と`sandboxName`を指定します。

### 一般的なJSON設定 {#mcp-connect-json}

JSON ベースのMCP サーバー設定（Claude Desktop （`claude_desktop_config.json`）、VS Code、または`mcp.json` ファイルを読み取る任意のクライアントなど）を受け入れるクライアントの場合、クライアントがネイティブリモート HTTPをサポートしているか、ローカルブリッジを必要とするかに応じて、次のいずれかの形式を使用します。

**`mcp-remote` ブリッジ経由（Claude Desktopおよびローカル ブリッジを必要とするその他のクライアント）**

```json
{
  "mcpServers": {
    "rtcdp": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://rtcdp-mcp.adobe.io/mcp"
      ]
    }
  }
}
```

**ネイティブ リモート HTTP （直接サポートするクライアント）**

```json
{
  "mcpServers": {
    "rtcdp": {
      "url": "https://rtcdp-mcp.adobe.io/mcp",
      "transport": "http"
    }
  }
}
```

>[!NOTE]
>
>設定にAPI キー、ベアラートークン、または追加のヘッダーは必要ありません。 認証は、初回使用時に、ブラウザベースのAdobe サインインフローを通じて完全に処理されます。

### UI ベースのクライアントへのインストール {#mcp-connect-ui}

#### クロード

`claude.ai`とClaude Desktopの場合、Real-Time CDP MCP サーバーを&#x200B;**カスタムコネクタ**&#x200B;としてサーバーURL `https://rtcdp-mcp.adobe.io/mcp`を使用して追加します。

* **個別プラン** — Claudeで、**→ コネクタのカスタマイズ**&#x200B;に移動し、**コネクタの追加**&#x200B;を選択して、サーバーURLを入力します。
* **グループ版およびエンタープライズ版プラン** — ワークスペース **オーナー**&#x200B;または&#x200B;**プライマリオーナー**&#x200B;がコネクタを&#x200B;**Organization settings → Connectors**&#x200B;の下に追加します。 追加すると、各ユーザーが独自のクラウド設定で有効になります。

コネクタを追加したら、会話でコネクタを有効にし、最初の使用時にAdobe ブラウザーログインを完了します。 ClaudeはAdobe認証サーバーを自動的に検出します。クライアント IDまたはクライアントシークレットは必要ありません。

#### ChatGPT

[!DNL ChatGPT]で、Real-Time CDP MCP サーバーを&#x200B;**カスタムコネクタ**&#x200B;として追加します。

1. **設定→コネクタ** （プランに応じて&#x200B;**設定→アプリとコネクタ**）に移動します。
2. 「**コネクタを追加**」を選択し、サーバーURLとして「`https://rtcdp-mcp.adobe.io/mcp`」と入力します。
3. コネクタを保存します。 [!DNL ChatGPT] プランによっては、この手順では&#x200B;**開発者モード**&#x200B;またはワークスペース管理者の承認が必要になる場合があります。
4. コネクタを有効にしたら、最初の使用時にプロンプトが表示されたら、Adobe ブラウザーログインを使用して認証します。

#### カーソル

Cursorで、Real-Time CDP MCP サーバーをリモート MCP サーバーとして追加します。

1. MCP **の**&#x200B;設定→開きます。
2. 「**新しいサーバーを追加**」を選択し、サーバーURLとして「`https://rtcdp-mcp.adobe.io/mcp`」と入力します。
3. 「**connect**」を選択して、ブラウザーベースのAdobe ログインをトリガーし、認証します。

接続すると、Real-Time CDP ツールはCursorのコンポーザーモードとエージェントモードで使用できるようになります。

#### その他のUI ベースのクライアント

VS Codeなどのクライアントや、リモート MCPをサポートする他のデスクトップおよびweb アプリケーションの場合は、`https://rtcdp-mcp.adobe.io/mcp`を使用してReal-Time CDP MCP サーバーを&#x200B;**リモート HTTP** サーバーとして追加します。 クライアントがオプションのヘッダーまたはベアラートークンをサポートしている場合は、それらを空のままにします。認証は、初回使用時にブラウザベースのAdobe サインインフローを通じて処理されます。

### テクニカルクライアントへのインストール {#mcp-connect-technical}

#### クロード・コード

ターミナルからサーバーを追加します。

```bash
claude mcp add --transport http rtcdp https://rtcdp-mcp.adobe.io/mcp
```

次に、Claude Codeを起動し、以下を実行します。

```text
/mcp
```

`rtcdp` サーバーを選択し、ブラウザーでAdobe ログインフローを実行します。 `claude.ai`でサーバーを既に追加している場合、両方が同じアカウントにサインインすると、同じサーバーが自動的にClaude Codeに表示される場合があります。

#### Codex

ターミナルからサーバーを追加します。

```bash
codex mcp add rtcdp --url https://rtcdp-mcp.adobe.io/mcp
```

サーバーの認証：

```bash
codex mcp login rtcdp
```

設定を確認します。

```bash
codex mcp list
```

サーバーを`~/.codex/config.toml`に直接追加することもできます。

```toml
[mcp_servers.rtcdp]
url = "https://rtcdp-mcp.adobe.io/mcp"
```

### 必須のリクエストパラメーター {#mcp-connect-params}

すべてのツール呼び出しには、Adobe Experience Platform テナントにリクエストをスコープする2つのパラメーターが必要です。

* `imsOrgId` – 下流のExperience Platform API呼び出しで`x-gw-ims-org-id` ヘッダーにマッピングされた組織ID。
* `sandboxName` — `x-sandbox-name` ヘッダーにマッピングされたExperience Platform サンドボックス名。

各セッションの開始時に提供してください。 次に例を示します。

> 「このセッションには、組織`1234ABCD@AdobeOrg`とサンドボックス `prod`を使用します。」

組織IDがわからない場合は、AI アシスタントに`search_organizations`への電話を依頼します。Adobeの資格情報がアクセス可能なすべての組織が返されます。

## 既知の制限事項（Beta） {#mcp-limitations}

次の制限は、[!DNL Adobe Real-Time CDP] MCP サーバーの現在のBeta リリースに適用されます。

| 制限事項 | 説明 | 回避策 |
| --- | --- | --- |
| **読み取り専用サーフェス** | MCP サーバーは取得APIのみを公開します。 オーディエンス、宛先、またはデータフローを作成、更新、アクティベート、または削除することはできません。 | Real-Time CDP UIまたはExperience Platform REST APIを使用して書き込み操作を行います。 |
| **エンゲージメントまたは配信指標がありません** | MCP サーバーは、ダウンストリーム配信の統計、エンゲージメント、またはコンバージョン指標を宛先プラットフォームから返しません。 | エンゲージメントとコンバージョンのデータには、配信先プラットフォーム独自のレポート、Customer Journey Analytics MCP、Adobe Analytics MCPを使用できます。 |
| **セグメントクエリは外部で作成する必要があります** | `Preview Audience Membership`には入力として有効なPQLまたはSDD式が必要です。MCP サーバーはクエリを作成しません。 | セグメントビルダーUIまたはSegmentation Service APIを使用してPQL/SDD式を作成し、MCP プロンプトに貼り付けます。 |
| **継続トークンを使用したページネーション** | リストツールは、ページ分割された結果を返します。 非常に大きなサンドボックス全体で完全な列挙を行うには、`continuationToken`回の呼び出しを連鎖させる必要があります。 | リスト全体を列挙するのではなく、フィルター（名前、状態、接続仕様、時間範囲）を使用してクエリを絞り込みます。 |
| **アクティベーション実行のフィルタリングは時間ベースのみです** | `Inspect Activation Runs`は、ステータスと完了タイムスタンプ （epoch ms UTC）によるフィルタリングをサポートしていますが、エラータイプや宛先プラットフォームによるフィルタリングはサポートしていません。 | 最初に`flowId`でフィルターを実行し（`List Configured Destinations`から取得）、スコープを特定の宛先に実行します。 |
| セッション開始時に&#x200B;**組織IDが必要** | すべてのツール呼び出し（`search_organizations`を除く）には、明示的なパラメーターとして`imsOrgId`と`sandboxName`が必要です。 これらを指定しない場合、ツール呼び出しは失敗します。 | 各セッションの開始時に、AI アシスタントに「このセッションには組織`<YOUR_ORG_ID>`とサンドボックス `<SANDBOX_NAME>`を使用してください」と伝えます。 組織IDがわからない場合は、最初に`search_organizations`に電話してください。資格情報がアクセスできる組織が返されます。 |

## よくある質問 {#mcp-faq}

+++どのMCP クライアントがサポートされていますか？

Real-Time CDP MCP サーバーは、[!DNL Claude]、[!DNL ChatGPT]、[!DNL Claude Code]、[!DNL Codex]、[!DNL Cursor]および[!DNL VS Code]など、リモート MCP サーバーまたはカスタムコネクタをサポートするあらゆるクライアントで動作します。 セットアップフローはクライアントによって異なります。UI ベースのクライアントは通常、設定パネルまたはコネクターパネルからサーバーを追加しますが、Claude CodeやCodexなどのテクニカルクライアントはコマンドラインまたは設定ファイルからサーバーを追加できます。
+++

+++アクセス方法を教えてください。

アクセスは、Beta中にのみ招待状によって行われます。 登録をリクエストするには、Adobeのアカウント担当者（カスタマーサクセスマネージャー、テクニカルアカウントマネージャー、またはアカウントエグゼクティブ）にお問い合わせください。 Adobeの担当者がプロダクトチームと連携し、お客様のビジネスを支援します。 詳しくは、[ アクセスと有効化](#mcp-access)を参照してください。
+++

+++認証の仕組み？

認証は、**ブラウザーベースのログイン**&#x200B;を通じて処理されます。 MCP クライアントが最初にツールを呼び出すと、デフォルトのブラウザーが開き、Adobe ログインページが表示されます。 クライアントを認証および認証した後、セッションが確立され、後続のツール呼び出しがそれを再利用します。 API キーや長期間有効な資格情報をクライアント設定に保存する必要はありません。
+++

+++MCP経由でアクセスできるReal-Time CDP オブジェクトは何ですか？

オーディエンス、宛先タイプ、設定済みの宛先アカウント、宛先データフロー、ソース接続とターゲット接続、アクティベーションの実行履歴にアクセスできます。 操作は読み取り専用（取得API）です。書き込み操作は現在のリリースではサポートされていません。
+++

+++Real-Time CDP MCP サーバーを使用するには、開発者アクセス権が必要ですか？

いいえ。 MCP サーバーは、マーケティングとテクノロジーの両方のペルソナ向けに設計されています。 マーケターは、サポートされているMCP クライアントで自然言語プロンプトを使用してMCPと対話できます。一方、データエンジニアや開発者は、MCPをサポートする開発者ツールでMCPを使用できます。
+++
