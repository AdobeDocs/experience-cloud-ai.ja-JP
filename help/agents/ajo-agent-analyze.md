# ジャーニー分析エージェント：スキルの概要およびユーザーガイド

## 概要

Journey Agentを使用すると、Journey Optimizerのユーザーは、自然言語インターフェイスを使用してジャーニーを分析し、最適化できます。 Journey Agentを使用すると、実務担当者はスケジュールやオーディエンスの競合をすばやく特定して解決し、ジャーニー内のユーザーの放棄ポイントを検出して、インサイトやレコメンデーションを提供できます。 これにより、実務担当者は、データに基づく意思決定を行い、顧客エンゲージメントを向上させ、ジャーニーオーケストレーションを合理化できます。

>[!AVAILABILITY]
>
>Journey Agentは、AI アシスタントにアクセスできるすべてのお客様が利用できます。 ただし、Journey Agentの機能を完全に使用するには、次の権限が必要になります。
>
>**ジャーニーの表示**：この権限を持つユーザーは、Journey Agentを使用して AI アシスタントでジャーニーに関するインサイトを直接表示できます。
>>**ジャーニーの管理**：権限があれば、Journey Agentを使用して AI アシスタントで新しいジャーニーを直接作成できます。
>>**セグメントの表示**：この権限を持つユーザーは、Journey Agentを使用して AI アシスタントで直接オーディエンスに関するインサイトを表示できます。
>>**セグメントの管理**：権限があれば、Journey Agentを使用して AI アシスタントで新しいオーディエンスを直接作成できます。

## ユースケース

### Journey Agent Analyze の主なユースケース

Journey Agentの分析スキルには、マーケティング活動を最適化するために利用できる様々な機能が用意されています。

1. **ジャーニーフォールアウト分析**

   - ジャーニー中に顧客が離脱する場所と理由を特定します。
   - 離脱につながる顧客行動のパターンを検出します。
   - インサイトを使用してジャーニーのデザインを洗練させ、リテンションを向上させます。

1. **ジャーニーオーディエンスの重複分析**

   - 複数のジャーニーをまたいだオーディエンスの重複を分析します。
   - オーバーターゲティングに起因するオーディエンスの疲労を防ぐ。
   - セグメント化を最適化して、バランスの取れたエンゲージメントを確保します。

1. **ジャーニースケジュールの重複分析**

   - 同じオーディエンスをターゲティングするスケジュール済みジャーニー間のタイミングの競合を検出します。
   - 通信の過剰を避け、スケジュール効率を向上させます。
   - ジャーニーが最適な時間に実行されるようにすることで、オーディエンスへの影響を最大化します。

1. **オペレーショナルインサイト**

   - プロンプトベースのジャーニーインサイト – ジャーニーに関する運用インサイト、つまり「すべてのライブジャーニーを表示」を表示します。

エージェントは、これらの分析のそれぞれについて、問題を検出するだけでなく、問題を解決するための **実用的な推奨事項** も提供します。


## 範囲内および範囲外のスキル

### **範囲内**

Journey Agent Analyze では、次の機能がサポートされています。

- **事後対応クエリ**：ジャーニーのパフォーマンス、オーディエンスの使用状況、スケジュールの競合に関する具体的な質問をできるようにします。
- **他のエージェントとの統合**:Audience AgentおよびData Insights Agentと連携して、より詳細な分析を行います。
- **エージェントのレスポンス構造**：推論（ロジックの説明）、分析の概要（要点の強調）、問題の詳細（問題の説明）、推奨（次のステップの提案）。

### **範囲外**

次の機能は、現在サポートされていません。

- **ジャーニーの自動作成**
- **リアルタイム異常値検出**
- **チャネルの重複**
- **ジャーニー エントリ分析**
- **技術的問題分析**
- **疲労解析**

## ベストプラクティス

### ベストプラクティスの確認

Journey Agent Analyze の効果を最大限に高めるには、次のベストプラクティスに従います。

1. **具体的**：明確で簡潔なプロンプトを使用して、ターゲット設定されたインサイトを取得します。 例えば、「自分のジャーニーは何ですか？」と尋ねる代わりに、「先月に作成されたすべてのジャーニーをリストする」を指定します。
1. **インサイトの組み合わせ**:Audience AgentとData Insights Agentのインサイトを統合して、ジャーニーのパフォーマンスの全体像を把握します。
1. **反復的な絞り込み**：フォールアウト分析と重複分析を使用して、ジャーニーの設計とスケジュールを反復的に絞り込みます。


### 設定のベストプラクティス

- **明確な目標の定義**：ジャーニーを分析する前に、明確な目標を確立します（例：リテンションの向上、コンバージョンの増加）。
- **定期的な監視**：ジャーニーのパフォーマンスを定期的にレビューして、トレンドや異常値を特定するスケジュールを設定します。
- **セグメント化の最適化**：オーディエンスのセグメント化のバランスを取り、疲労を回避し、エンゲージメントを最大化します。
