# Data Engineering Agent

&lt;!—! – これはドラフトです – hidefromtocは、記事ではなくTOC.md ファイルで制御されるようになりました。 --->

Data Engineering Agentは、Agent Orchestratorを活用したAI ベースの副操縦士です。Adobe Experience Platformのデータチームを支援します。 このエージェントは、データモデリング、オンボーディング、SQL データの準備、ガバナンス、ライフサイクル管理など、エンドツーエンドのデータライフサイクル全体にわたる作業を処理するように設計されています。 Data Engineering Agentなら、エンジニアやアーキテクトは、データ品質を高め、手作業を削減しながら作業速度を向上できます。

Data Engineering Agentには、ワークフローを最適化するために活用できるスキルが揃っています。

| Data Engineering Agent skill | 説明 |
| --- | --- |
| データオンボーディング | <strong> データオンボーディング </strong>は、S3、Marketo、Data Landing Zone （DLZ）などのバッチソースをサポートしています。 機能には次のようなものがあります。 <ul><li>S3、Data Landing Zone、Marketoなど、様々なデータソースへの接続。</li><li>ソースデータのプロファイリング（例：異なる値、null、重複、基本的な品質指標）</li><li>ソースフィールドを標準XDM フィールドグループとカスタムフィールドの両方に整列させる。</li><li>データ取り込みデータフローの提案と構築（例：S3やMarketoからRTCDP/CJAのデータセットへのデータの移動）</li></ul> |
| 自動データ品質とセマンティックエンリッチメント | <ul><li>異常の特定や、有効/無効なレコード数のレポート作成など、包括的なデータ品質評価を実行。</li><li>タイプの正規化、通貨の書式設定、IDの重複排除など、データ品質の改善を推奨します。</li><li>データサンプリングとインテリジェントな命名パターン認識に基づいて、スキーマにセマンティックエンリッチメントを適用します。</li><li>推奨されるエンリッチメントについてユーザーに確認を促し、最終的で承認済みのデータ変更を確認できます。</li></ul> |

| データの検証 | |

Data Engineering Agentを活用することで、次のことを実現できます。

- **オンボーディングの迅速化**：数週間に及ぶ手動スキーママッピングとパイプライン設定から、ガイド付きの会話設定に要する時間を短縮します。
- **データ品質の向上**：組み込みのデータ品質分析とセマンティックチェックにより、無効なレコードと実稼動インシデントが少なくなります。
- **ガバナンスとコンプライアンスの改善**: ガバナンスポリシーは、後回しにするのではなく、デザイン時（ラベル、TTL、ID処理）に添付されます。
- **生産性の高いデータチーム**：反復的なSQL/データフロー作業を自動化し、エンジニアがより価値の高い設計と最適化に専念できるようにします。
- **より幅広いセルフサービス**：専門家でない関係者でも、一般的なデータ準備タスクをガードレールを使って安全にセルフサービスできます。
