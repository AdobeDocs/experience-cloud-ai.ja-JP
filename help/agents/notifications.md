# Notifications エージェント

通知エージェントは、[AI アシスタント ](../ai-assistant/ai-assistant-ui.md)内に会話型AI レイヤーを追加します。このレイヤーを使用すると、自然言語を使用して通知の要約、解釈、処理を行うことができ、これによりパッシブフィードがアクティブアシスタントに変換されます。 長いリストをスキャンする代わりに、「47の通知がある」から「金曜日までに承認が必要な2つのデータセット、競合がある3つのデータセット、残りはFYI」に1回の交換で移動できます。

## Notifications Agentの機能

Notifications Agentは次の機能をサポートしています。

| 機能 | 説明 |
| --- | --- |
| **通知の概要とインサイト** | 通知の概要を簡潔に説明し、アクションが必要な通知と、情報のみを目的とした通知を明確に区別して、通知の目的と受信した理由の両方を理解できます。 |
| **操作アクション** | 会話を通じて通知に対して直接アクションを起こす – それを既読としてマークしたり、一時停止させたり、個別またはグループ化された通知のリマインダーを設定したりします。 |
| **環境設定と設定** | 「設定」に移動することなく、ソリューションおよびカテゴリごとに通知チャネル（電子メール、Slack、アプリ内）を有効または無効にします。 |
| **ナビゲーションとディープリンク** | ディープリンクを介して、通知から関連商品エリアに直接移動します。 |
| **Collaborationと共有** | 同じ組織内の受信者に通知を共有したり、通知を送信したりすることができます。 |

## Access Notifications エージェント

Notifications Agentには、次の方法でアクセスできます。

- **AI アシスタントから**: AI アシスタントを開き、通知関連のプロンプトを入力します。
- **プロンプトライブラリ**&#x200B;から：要件に合った提案プロンプトを選択します。

## サンプルプロンプト

Notifications Agentで使用できるプロンプトの例については、次の節を参照してください。

### 要約とインサイト

- 「最近の通知を要約してください」
- 「自分のアクションが必要な通知とFYIが必要な通知はどれか？」

### アクション

- 「すべての実用的でない通知を読み取り済みとしてマークする」
- 「1時間後に{NOTIFICATION_OF_INTEREST}通知について通知を受け取ります。」

### 環境設定

- 「Journey OptimizerのSlack通知を有効にする」
- 「毎日のダイジェストを毎週に切り替える」

### コラボレーション

- 「この通知を{EMAIL_ADDRESS_OF_USER_IN_YOUR_ORG}と共有します。」

## ベストプラクティス

Notifications Agentを使用する際は、次のヒントに注意してください。

- **具体的であること** — エージェントが意図したターゲットに対してアクションを実行できるように、アプリ、通知、通知カテゴリ、チャネルに明確な名前を付けます。
- **チェーンプロンプト** – 概要を要求し、特定の通知に焦点を当ててフォローアップしてから、アクションを実行できます。これらはすべて、1回の会話で実行できます。
- **質問を明確にする回答** - エージェントがアプリ、チャネル、通知に関する確認を求める場合は、変更が正しく適用されるように必ず応答してください。
